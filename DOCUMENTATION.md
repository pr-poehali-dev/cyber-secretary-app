# LexDesk — Документация

## Что такое LexDesk

LexDesk — веб-приложение для управления адвокатской практикой. Позволяет вести базу доверителей, планировать день, отслеживать следственные действия, контролировать сроки обжалования, формировать ходатайства на оплату и анализировать практику.

---

## Разделы приложения

| Раздел | Что делает |
|---|---|
| Рабочий стол | Статистика дня, срочные задачи, расписание |
| Планирование дня | Создание и отметка задач по дням |
| Доверители | База клиентов с делами, следователями, историей |
| Ходатайства | Формирование ходатайства на оплату (PDF / DOC) |
| Сроки обжалования | Контроль дат апелляций и жалоб |
| Следственные действия | Список следственных действий с фильтрацией |
| Аналитика | Доходы, категории дел, статистика |
| Настройки | Профиль, уведомления, правила сроков |

## Система доступа (подписки)

### Статусы пользователя

| Статус | Что означает |
|---|---|
| `trial` | Пробный период (по умолчанию 10 дней с момента регистрации) |
| `active` | Оплаченная подписка, действует до указанной даты |
| `free` | Бессрочный бесплатный доступ (для партнёров, администратора) |
| `blocked` | Доступ принудительно закрыт администратором |

### Как работает

1. При регистрации пользователь получает пробный период (10 дней).
2. После истечения пробного периода — экран оплаты с реквизитами.
3. Пользователь переводит деньги и нажимает «Сообщить об оплате».
4. Администратор видит заявку в панели и меняет статус на `active` с датой окончания.
5. Пользователь сразу получает доступ.

### Администратор

Пользователь `mayckova@mail.ru` имеет статус `free` (вечный бесплатный) и права администратора.

В сайдбаре у администратора появляется пункт **«Администрирование»** с двумя вкладками:
- **Пользователи** — список всех пользователей, редактирование статуса, дат, заметок
- **Настройки оплаты** — цена, срок подписки, реквизиты для оплаты

## Панель администратора

### Управление пользователями

Кликните на пользователя, чтобы открыть форму редактирования:

- **Статус** — trial / active / free / blocked
- **Оплачено до** — дата (только при статусе `active`)
- **Пробных дней** — сколько дней пробного периода (можно продлить)
- **Заметка** — внутренняя заметка (пользователь не видит)

### Настройки оплаты

- **Цена подписки** (₽) — отображается пользователю на экране блокировки
- **Срок подписки** (дней) — используется при формировании даты истечения
- **Реквизиты / инструкция** — текст, который видит пользователь при оплате

## Развёртывание на реальном домене

### Шаг 1 — Выбор домена

Зарегистрируйте домен у любого регистратора (REG.RU, Timeweb, Beget и др.).

### Шаг 2 — Подключение в poehali.dev

1. Откройте ваш проект на [poehali.dev](https://poehali.dev)
2. Нажмите **«Опубликовать»** → **«Привязать свой домен»**
3. Введите домен (например: `lexdesk.ru`)
4. Получите DNS-записи: нужно добавить **CNAME** или **A-запись** у регистратора
5. SSL-сертификат подключается автоматически

Подробнее: [docs.poehali.dev/deploy/publish](https://docs.poehali.dev/deploy/publish)

### Шаг 3 — Проверка

После обновления DNS (обычно 1–24 часа) сайт будет доступен на вашем домене с HTTPS.

## Как дорабатывать сайт

### Инструменты

| Инструмент | Для чего |
|---|---|
| [poehali.dev](https://poehali.dev) | Основная платформа — ИИ-редактор, деплой, домены |
| GitHub | Версионирование кода (Скачать → Подключить GitHub) |
| VS Code / Cursor | Локальная разработка |

### Стек технологий

- **Фронтенд**: React + TypeScript + Vite + Tailwind CSS
- **UI-компоненты**: shadcn/ui (`src/components/ui/`)
- **Бэкенд**: Python 3.11 Cloud Functions (`/backend/`)
- **База данных**: PostgreSQL (схема `t_p21225702_cyber_secretary_app`)
- **Авторизация**: JWT-токены в localStorage, сессии в БД

### Структура проекта

```
src/
  pages/           — все страницы и разделы приложения
    Index.tsx      — главный контейнер с навигацией
    AuthPage.tsx   — вход / регистрация
    DashboardPlanning.tsx — рабочий стол и планирование
    ClientDetailPanel.tsx — доверители
    PetitionsSection.tsx  — ходатайства
    DeadlinesSection.tsx  — сроки
    InvestigationsSection.tsx — следственные действия
    AnalyticsSection.tsx  — аналитика
    Settings.tsx   — настройки
    AdminPanel.tsx — панель администратора
    SubscriptionWall.tsx  — экран блокировки
  api.ts           — все запросы к бэкенду
  auth.ts          — авторизация

backend/
  api/             — основные CRUD-операции (клиенты, задачи, дела, сроки)
  auth/            — регистрация, вход, профиль
  billing/         — подписки и администрирование
  notify/          — уведомления по email
  func2url.json    — URLs всех функций

db_migrations/     — SQL-миграции базы данных
```

### Добавление нового раздела

1. Создайте файл `src/pages/MySection.tsx` с компонентом `export function MySection()`
2. Добавьте раздел в `types-and-data.ts` в тип `Section`
3. Добавьте пункт в массив `navItems` в `Index.tsx`
4. Добавьте `case "my-section": return <MySection />` в `renderSection()`

### Изменение базы данных

1. Напишите SQL-миграцию
2. Выполните её через инструмент `migrate_db` (или через ИИ-редактор поехали)
3. Файл автоматически сохранится в `db_migrations/`

### Добавление бэкенд-функции

1. Создайте папку `backend/my-function/`
2. Добавьте `index.py` с функцией `handler(event, context)`
3. Добавьте `requirements.txt` и `tests.json`
4. Вызовите `sync_backend` — функция деплоится автоматически
5. URL появится в `backend/func2url.json`

## FAQ

**Как войти в систему администратора?**  
Войдите как `mayckova@mail.ru`. В левом меню появится пункт «Администрирование».

**Как дать пользователю бесплатный доступ?**  
В панели администратора → Пользователи → кликните на пользователя → Статус: «Бесплатный (free)» → Сохранить.

**Как заблокировать пользователя?**  
В панели администратора → Пользователи → кликните на пользователя → Статус: «Заблокирован (blocked)» → Сохранить.

**Как продлить пробный период?**  
В форме редактирования пользователя увеличьте поле «Пробных дней» — отсчёт идёт с момента регистрации.

**Как изменить цену или реквизиты?**  
Администрирование → вкладка «Настройки оплаты» → измените нужные поля → Сохранить настройки.

**Как скачать код сайта?**  
На платформе poehali.dev: **Скачать → Скачать код** (ZIP) или **Скачать → Подключить GitHub**.

**Как получить HTML-билд для хостинга?**  
На платформе poehali.dev: **Скачать → Скачать билд** — готовый HTML+JS+CSS.
